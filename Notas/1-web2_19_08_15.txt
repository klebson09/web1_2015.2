E a Solicitação?
GET -> só obtem a resposta não alterando o estado do servidor
1 usuario digita um site
2 o servidor busca a página (caso a página nnão esteja lá ele retorna com um erro 200)

POST ->  solicita algo e ao mesmo tempo envia dados (ex. um formulário)

na mensagem de texto GET e POST
até a interrogação é a informação 

obs.
color=dark&teste=malty
GET- esse termo no inicio da mensagem está exposto no URL (obs. é perigoso por ficar visivel - usuario e senha) o GET tem um tamanho limitado
POST - esse termo no fim da mensagem está encapsulada no corpo
o resultado vai ser sempre um HTML

quando usar GET e POST?
GET
-solicitar uma página via hiperlink

POST
-digitar login e senha
-enviar um formulário para o servidor
-enviar uma resposta
OBS.
GET obter coisas
POST enviar dados para serem processados
